# Changelog

Все значимые изменения в проекте документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
и проект придерживается [Semantic Versioning](https://semver.org/lang/ru/).

## [1.4.0] - 2026-05-31

### Added
- Поиск по переводчикам: возможность искать книги по имени переводчика через отдельный режим поиска
- Индексы полнотекстового поиска для таблицы переводчиков (`zz_40_fill_FT.sql`)
- Конфигурация скриптов обновления БД вынесена в отдельные файлы (`scripts.conf`, `tasks.conf`)

### Fixed
- Снижение потребления ресурсов при работе бота

## [1.3.2] - 2026-05-24

### Fixed
- Исправлено логирование названия книги при скачивании: вместо ID книги и формата (`12345.fb2`) теперь логируется реальное имя файла из заголовка `Content-Disposition` ответа сервера Flibusta (например, `Толстой - Война и мир.fb2`)

## [1.3.1] - 2026-05-24

### Fixed
- Исправлено логирование поискового запроса в `structured_logger.log_search()`: при показе популярных книг вместо текста запроса записывался `None`, а при обычном поиске — значение переключателя. Теперь корректно логируется текст запроса, значение переключателя популярных книг или `"popular"` как fallback.

## [1.3.0] - 2026-05-21

### Added
- Админ-панель: кнопка «📢 Рассылка» для массовой отправки сообщений всем пользователям
- Режим тестовой рассылки через переменные окружения (`BROADCAST_TEST_ONLY`, `BROADCAST_TEST_USER_IDS`)
- Подтверждение рассылки через inline-кнопки (отправить / отмена)
- Логирование результатов рассылки в `StructuredLog` (новые типы событий `broadcast.sent`, `broadcast.failed`, `broadcast.started`)
- Метод `get_all_user_ids()` в `DatabaseSettings` для получения списка всех пользователей
